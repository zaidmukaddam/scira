# AI Issues

Bug reports, known issues, and tracking for the Scira self-hosted instance.

## Purpose

Track bugs, issues, and blockers discovered during development and usage.

## File Organization

- **Active Issues** - Open bugs and blockers (prefix with `ISSUE_`)
- **Resolved Issues** - Archived for reference (move to `resolved/` subdirectory)
- One file per major issue

## Current Documents

- None yet (directory just created)

## Issue Template

When creating new issue files, include:
1. **Title** - Clear, concise description
2. **Status** - Open, In Progress, Resolved
3. **Priority** - Critical, High, Medium, Low
4. **Description** - Detailed explanation of the issue
5. **Steps to Reproduce** - If applicable
6. **Expected vs Actual Behavior**
7. **Root Cause** - If identified
8. **Solution** - Once resolved
9. **Related Files** - Links to affected code
10. **Date Opened/Resolved**

## Guidelines

- Keep issues updated with current status
- Archive resolved issues promptly
- Link to commits that fix issues
- Include screenshots/logs where helpful
